# /thesis-check

> 주가 하락 시 "thesis가 여전히 유효한가?" 빠른 점검

## 목표

**흔들릴 때 빠르게 확인**:
1. 핵심 thesis가 **깨졌는가?**
2. 새로운 **치명적 리스크**가 발생했는가?
3. 지금 해야 할 **액션**은 무엇인가?

## 언제 사용하나?

| 상황 | 사용 |
|------|------|
| 주가가 -10% 이상 급락 | ✓ |
| 부정적 뉴스/루머 발생 | ✓ |
| 실적 발표 후 급변동 | ✓ (또는 /earnings-flash) |
| 시장 전체 급락으로 불안 | ✓ |
| 그냥 정기 점검 | ✓ (월 1회 권장) |

---

## 입력

```
/thesis-check [티커] (이유)
```

**전제 조건:** `research/[TICKER]/thesis_*.md` 파일이 존재해야 함

**예시:**
```
/thesis-check IREN
/thesis-check IREN 오늘 -15% 빠졌어
/thesis-check IREN 희석 공시 나왔어
/thesis-check PLTR 금리 인상 뉴스
```

---

## 점검 플로우

```
[1단계: Thesis 로드]
    │  research/[TICKER]/thesis_*.md 핵심 포인트 추출
    │  - 투자 논리 (3~5개)
    │  - 리스크 (인지했던 것)
    │  - 매매 기준
    ▼
[2단계: 현재 상황 스캔]
    │  - 최근 뉴스/공시 확인
    │  - 주가/거래량 변화
    │  - 섹터/시장 전체 상황
    ▼
[3단계: 논리별 점검]
    │  각 투자 논리가:
    │  ✓ 유효 / ⚠️ 약화 / ✗ 붕괴
    ▼
[4단계: 판정]
    │  HOLD / WATCH / EXIT 중 하나
    ▼
저장: research/[TICKER]/check_YYYY-MM-DD.md
```

## 점검 항목

### 핵심 질문 3가지

| 질문 | 확인 사항 |
|------|----------|
| **1. Thesis 깨졌나?** | 핵심 투자 논리가 무효화됐는가? |
| **2. 새 리스크 생겼나?** | 기존에 없던 치명적 위험 발생? |
| **3. 유동성 바뀌었나?** | 자금 흐름에 구조적 변화? |

### 체크리스트


## 판정 기준

### HOLD (유지)

```
조건:
- 핵심 논리 모두 유효 (✓)
- 신규 치명적 리스크 없음
- 주가 하락이 펀더멘탈과 무관

액션:
- 아무것도 하지 않음
- (옵션) 추가 매수 기회로 검토
```

### WATCH (관찰)

```
조건:
- 핵심 논리 중 1~2개 약화 (⚠️)
- 또는 신규 리스크 발생 (심각도 중간)
- 아직 붕괴는 아니지만 모니터링 필요

액션:
- 추가 매수 중단
- 모니터링 포인트 설정
- 다음 점검 일정 잡기
```

### EXIT (재검토/청산)

```
조건:
- 핵심 논리 중 1개 이상 붕괴 (✗)
- 또는 신규 치명적 리스크 현실화
- 또는 thesis 전체 재검토 필요

액션:
- 즉시 비중 축소 또는 청산 검토
- /deep-dive로 전면 재분석
- 새 thesis 필요
```

시간이 없을 때 3분 내 핵심만 확인:

```
사용자: "/thesis-check IREN 빠르게"

Claude:
## IREN 빠른 점검 (YYYY-MM-DD)

**트리거**: (입력한 이유 또는 추정)

**핵심 3줄 점검**:
1. Thesis 깨졌나? → 아니오 ✓
2. 새 치명적 리스크? → 아니오 ✓
3. 유동성 변화? → 아니오 ✓

**판정**: HOLD

**한 줄 요약**: 펀더멘탈 변화 없음, 시장 노이즈로 판단
```

## 흔들림 대응 가이드

### 주가 -20% 급락 시

```
1. 일단 아무것도 하지 않는다 (패닉 금지)
2. /thesis-check 실행
3. 판정이 HOLD면:
   - "thesis가 유효하면 이건 바겐세일"
   - 추가 매수 여력 확인
4. 판정이 WATCH/EXIT면:
   - 감정적 결정 금지
   - 24시간 후 재점검
```

### 부정적 뉴스 발생 시

```
1. 뉴스의 실체 확인 (루머 vs 팩트)
2. /thesis-check [티커] [뉴스 요약]
3. thesis에 직접 영향 여부 확인
4. 직접 영향 없으면 노이즈로 무시
```

### 시장 전체 급락 시

```
1. 개별 기업 문제인지 확인
2. 시장 급락이면 thesis와 무관
3. /thesis-check로 확인 후 HOLD
4. (옵션) 현금 있으면 추가 매수 기회
```
