# 질문 가이드 (단계별)

## 1단계: 관심의 시작 (빈 캔버스)

**목적**: 사전 판단 없이 사용자의 출발점 파악

```markdown
- "이 기업에 처음 관심을 가지게 된 계기가 뭔가요?"
- "어디서 처음 알게 되셨나요? (뉴스, 유튜브, 지인 등)"
- "처음 봤을 때 뭐가 눈에 들어왔나요?"
- "그 때 어떤 느낌이었나요?"
```

---

## 2단계: 시대의 병목 (내러티브)

**목적**: 기업이 해결하는 시대적 문제 정의

```markdown
- "이 기업이 해결하려는 문제가 뭐라고 보세요?"
- "왜 이 문제가 '지금' 중요해졌나요?"
- "이 문제가 해결 안 되면 세상에 어떤 일이 생기나요?"
- "5년 후에도 이 문제가 여전히 중요할까요?"
- "이 기업 말고 이 문제를 해결할 수 있는 곳이 있나요?"
```

---

## 3단계: 유동성 환경

**목적**: 거시 환경과 자금 흐름 인식

```markdown
- "지금 이 섹터에 돈이 몰리고 있다고 보세요?"
- "왜 돈이 여기로 흘러온다고 생각하세요?"
- "연준이 금리를 올리면/내리면 이 기업에 어떤 영향이 있을까요?"
- "달러 강세/약세가 이 기업에 어떻게 작용할까요?"
```

---

## 4단계: 경쟁 우위

**목적**: 지속 가능한 강점 파악

```markdown
- "이 기업만이 가진 강점이 뭐라고 보세요?"
- "경쟁사들이 왜 못 따라오나요?"
- "이 강점이 3년 후에도 유효할까요?"
- "만약 경쟁사가 10배 자본을 투입하면 따라잡을 수 있나요?"
```

---

## 5단계: 리스크 인식

**목적**: 실패 시나리오 명확화

```markdown
- "이 투자가 망한다면 어떤 이유일까요?"
- "가장 걱정되는 점이 뭔가요?"
- "최악의 시나리오는 뭐라고 생각하세요?"
- "그 리스크를 알면서도 투자하려는 이유가 뭔가요?"
- "얼마나 떨어지면 손절하실 건가요? 왜 그 숫자인가요?"
```

---

## 6단계: 타이밍

**목적**: 진입/청산 기준 명확화

```markdown
- "왜 '지금' 사야 한다고 생각하세요?"
- "6개월 기다리면 더 좋은 가격에 살 수 있지 않을까요?"
- "어떤 신호가 오면 추가 매수하실 건가요?"
- "어떤 일이 생기면 파실 건가요?"
```

---

## 7단계: Thesis 정리

**목적**: 사용자 언어로 thesis 구조화

```markdown
지금까지 말씀하신 내용을 정리해보면:

## [기업명] 투자 Thesis

### 핵심 논리
- (사용자 답변 기반 정리)

### 시대의 병목
- (사용자 답변 기반 정리)

### 유동성 판단
- (사용자 답변 기반 정리)

### 경쟁 우위
- (사용자 답변 기반 정리)

### 인식한 리스크
- (사용자 답변 기반 정리)

### 매매 기준
- 추가 매수 신호: (사용자 답변)
- 청산 신호: (사용자 답변)

---

"이 정리가 맞나요? 수정하거나 추가할 부분 있으세요?"
```

## 파고들기 질문

사용자 답변이 모호할 때 사용:

```markdown
- "조금 더 구체적으로 말씀해주실 수 있나요?"
- "예를 들면 어떤 상황인가요?"
- "그게 왜 중요하다고 생각하세요?"
- "다르게 표현하면 어떻게 될까요?"
```
