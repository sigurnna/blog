# 코치 플로우 상세

## 전체 흐름

```
[1단계: 관심의 시작]
    │  "이 기업에 왜 관심을 가지셨나요?"
    │  "처음 어디서 알게 되셨나요?"
    │  "무엇이 눈에 들어왔나요?"
    ▼
[2단계: 시대의 병목]
    │  "이 기업이 해결하는 문제가 뭐라고 보세요?"
    │  "왜 지금 이 문제가 중요해졌나요?"
    │  "이 문제가 해결 안 되면 어떤 일이 생기나요?"
    ▼
[3단계: 유동성 환경]
    │  "지금 돈의 흐름은 어떻게 보세요?"
    │  "이 섹터에 돈이 몰리는 이유가 뭘까요?"
    │  "금리/유동성 환경이 바뀌면 어떻게 될까요?"
    ▼
[4단계: 경쟁 우위]
    │  "이 기업만의 강점이 뭐라고 보세요?"
    │  "경쟁사가 따라오기 어려운 이유가 있나요?"
    │  "3년 후에도 이 강점이 유효할까요?"
    ▼
[5단계: 리스크 인식]
    │  "이 투자가 실패한다면 어떤 이유일까요?"
    │  "가장 걱정되는 부분이 뭔가요?"
    │  "그 리스크를 감수할 수 있는 이유는요?"
    ▼
[6단계: 타이밍]
    │  "왜 지금이 적기라고 생각하세요?"
    │  "더 기다리면 어떤 일이 생길까요?"
    │  "어떤 신호가 오면 더 사거나 팔 건가요?"
    ▼
[7단계: Thesis 정리]
    │  사용자의 답변을 바탕으로 thesis 초안 제시
    │  "이게 맞나요? 수정할 부분 있으세요?"
    ▼
저장: reports/[TICKER]/thesis.md
```

---

## 단계별 목적

| 단계 | 목적 | 핵심 질문 |
|------|------|----------|
| 1. 관심의 시작 | 사전 판단 없이 출발점 파악 | "왜 관심?" |
| 2. 시대의 병목 | 시대적 문제 정의 | "무슨 문제 해결?" |
| 3. 유동성 환경 | 거시 환경 인식 | "돈의 흐름?" |
| 4. 경쟁 우위 | 지속 가능한 강점 | "왜 이 기업?" |
| 5. 리스크 인식 | 실패 시나리오 명확화 | "망하면 왜?" |
| 6. 타이밍 | 진입/청산 기준 | "왜 지금?" |
| 7. Thesis 정리 | 사용자 언어로 구조화 | 정리 + 확인 |

상세 질문: [질문 가이드](questions.md)

## 사용 시나리오

### 기본 사용

```
사용자: "/coach IREN"

Claude: "IREN에 관심을 가지게 된 계기가 뭔가요?"

사용자: (답변)

Claude: (다음 질문)

... (대화 진행) ...

Claude: "지금까지 말씀하신 내용을 정리해보면..."
Claude: "저장할까요?"
```

### 기존 thesis 수정

```
사용자: "/coach IREN 수정"

→ reports/IREN/thesis.md 로드
→ 기존 내용 확인 후 수정할 부분 질문
→ 동일 파일에 업데이트
```
